# PÁN KEBAB — сайт + адмінка

Статичний сайт з меню (Düriim, Döner, Box, Tanier, Baklava), контактами й картою,
плюс захищена паролем адмінка `/admin` для редагування назв/описів/цін/фото без
доступу до коду. Дані живуть у Supabase, авторизація і запис відбуваються через
Netlify Functions — пароль ніде не зберігається у відкритому вигляді.

## Структура проєкту

```
index.html              головна сторінка
css/style.css           стилі сайту
js/main.js               завантажує меню, рахує "відкрито/закрито"
js/menu-data.js          запасні дані, якщо бекенд ще не підʼєднано
assets/mascot.svg        заглушка банера-персонажа (замініть на оригінал)
assets/img/*.svg         заглушки фото страв
admin/                   адмінка (логін + редактор меню)
netlify/functions/       серверні функції (логін, читання/запис меню, фото)
supabase/schema.sql      SQL для створення таблиці й політик безпеки
scripts/hash-password.js утиліта для зміни пароля
.env.example             перелік потрібних env-змінних (без реальних значень)
```

## Крок 1. Supabase (база даних + сховище фото)

1. Зареєструйтесь на [supabase.com](https://supabase.com) (безкоштовно) і створіть новий проєкт.
2. Зайдіть у **SQL Editor → New query**, вставте весь вміст файлу `supabase/schema.sql` і натисніть **Run**. Це створить таблицю `menu_items`, увімкне захист (RLS — писати може тільки сервер, а не будь-який відвідувач), створить сховище `menu-images` і одразу заповнить меню п'ятьма стартовими позиціями.
3. У **Project Settings → API** скопіюйте:
   - `Project URL` → це буде `SUPABASE_URL`
   - `service_role` ключ (не `anon`!) → це буде `SUPABASE_SERVICE_ROLE_KEY`. Це секретний ключ з повним доступом — він піде тільки в Netlify, ніколи в код чи GitHub.

## Крок 2. GitHub

```bash
cd pan-kebab
git init
git add .
git commit -m "PÁN KEBAB — перший варіант сайту"
git branch -M main
git remote add origin https://github.com/ВАШ-АКАУНТ/pan-kebab.git

```

`.gitignore` вже виключає `node_modules` і `.env` — реальні секрети в репозиторій не потраплять.

## Крок 3. Netlify

1. На [netlify.com](https://app.netlify.com) → **Add new site → Import an existing project** → виберіть щойно створений GitHub-репозиторій.
2. Netlify сам підхопить налаштування з `netlify.toml` (publish = `.`, functions = `netlify/functions`). Окрема build-команда не потрібна — це статичний сайт.
3. Перед першим деплоєм (або відразу після) зайдіть у **Site configuration → Environment variables** і додайте чотири змінні з файлу `.env.example`:
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `JWT_SECRET` — будь-який довгий випадковий рядок (можна згенерувати командою `openssl rand -hex 32`)
   - `ADMIN_PASSWORD_HASH` — можна залишити значення з `.env.example` (це хеш пароля `obsluha123`), або одразу поставити свій (Крок 5).
4. Натисніть **Deploy site**. Через хвилину сайт буде доступний на `https://ваш-сайт.netlify.app`.

## Перевірка

- Головна сторінка: меню має показати 5 позицій із заглушками фото.
- `/admin`: введіть пароль `obsluha123` (якщо не змінювали хеш) — повинен відкритись список страв з полями для редагування.
- Збережіть будь-яку зміну (наприклад, ціну) і перезавантажте головну сторінку — зміна має з'явитись там за кілька секунд.

## Зміна пароля адмінки

Пароль ніде не зберігається у відкритому вигляді — тільки його bcrypt-хеш в env-змінній. Щоб поставити новий пароль:

```bash
node scripts/hash-password.js "новийПарольТут"
```

Скрипт виведе рядок `ADMIN_PASSWORD_HASH=...` — скопіюйте значення в Netlify env variables замість старого і натисніть **Redeploy** (зміна env-змінної сама по собі не рестартує функції, потрібен повторний деплой).

## Заміна банера й фото страв

- **Банер-персонаж**: я не маю доступу до оригінального зображення, яке заливалось у попередньому чаті, тож зараз стоїть SVG-заглушка в тому ж дусі (червоний фон, біла обводка). Щоб поставити оригінал — просто замініть файл `assets/mascot.svg` на своє зображення з такою ж назвою (або зміните шлях у `index.html`, рядок з `<img class="hero-mascot" ...>`).
- **Фото страв**: найпростіше — через `/admin`, кнопка "Змінити фотку" під кожною позицією (jpg/png/webp, до 5 МБ). Фото одразу зберігається в Supabase Storage і підмінює заглушку на сайті.

## Важливі обмеження безкоштовних тарифів

- **Supabase free**: проєкт призупиняється після ~1 тижня без жодних звернень до API. Якщо на сайт хоч раз на тиждень хтось заходить (а саме завантаження головної сторінки звертається до бази), пауза не настане. Якщо ж проєкт призупинився — досить зайти в Supabase dashboard і натиснути "Restore project".
- **Netlify free**: ліміти на кількість викликів функцій і трафік на місяць більш ніж достатні для одного кебабного прилавка; перевищення малоймовірне.

## Локальний перегляд без бекенду

Якщо просто відкрити `index.html` у браузері до підключення Supabase/Netlify — сайт покаже заглушкові дані з `js/menu-data.js`, щоб одразу бачити верстку. Адмінка ж потребує задеплоєних функцій і запрацює тільки після Кроку 3.
